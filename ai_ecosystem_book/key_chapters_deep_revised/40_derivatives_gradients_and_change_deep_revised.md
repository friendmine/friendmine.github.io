# 第40章 导数、梯度与变化

上一章讨论统计与数据，说明模型的学习发生在具体样本和数据分布中。第40章进入训练机制本身：当模型在样本上犯错之后，系统如何知道参数应该朝哪个方向改变。

深度学习能够训练，依靠的是损失函数、导数、梯度和链式法则共同把“错误如何随参数变化”变成可计算信号。这个信号不会直接给出答案，但它能告诉系统在当前位置附近，哪些方向更像是在降低错误。

这也是从“模型看到了数据”到“模型开始改变参数”的第一道门。没有梯度，训练会退回盲目搜索；有了梯度，系统才有机会把一次次局部修正组织成长期能力。

## 1. 训练需要知道错误如何随参数变化

一个模型初始化之后，参数通常只是某种随机起点。它可以产生输出，也可以计算损失，但这还不够。训练更需要知道：当某个参数轻微改变时，损失会怎样变化。

如果系统只知道“当前错了多少”，却不知道“怎样改变会让错误下降”，它就只能靠盲目试探。参数空间维度极高，盲目试探缺少工程可行性。

于是，训练问题被改写成一个更具体的问题：

- 当前损失是多少
- 损失对每个参数有多敏感
- 哪些方向会让损失上升
- 哪些方向更可能让损失下降
- 这些方向信号是否足够稳定，可以支持反复更新

导数和梯度的作用，就是把这些变化关系变成可以计算、可以传递、可以用于更新参数的信号。

## 2. 导数刻画的是局部敏感度

导数常被讲成极限和符号规则，但在训练语境里，它首先回答一个务实问题：输入或参数发生很小变化时，输出或损失会变化多少。

最基本的定义可以写成：

```text
f'(x) = lim(h -> 0) (f(x + h) - f(x)) / h
```

这个式子描述的是局部敏感度。如果某个参数轻微变化，损失剧烈变化，说明那一带对这个参数很敏感；如果参数变化后损失变化很小，说明这个方向上的信号较弱。

对机器学习来说，导数不是数学装饰，而是训练系统对局部地形的一次测量。它不会告诉系统全局终点在哪里，却会告诉系统当前位置附近的变化趋势。

## 3. 梯度把许多局部敏感度组织成方向信号

真实模型有大量参数。一个参数的导数只能描述一个方向上的变化，无法说明整个模型应该怎样更新。梯度把所有参数方向上的偏导数组织成一个向量。

常见写法是：

```text
∇L(θ) = [∂L/∂θ1, ∂L/∂θ2, ..., ∂L/∂θn]
```

这里 `L(θ)` 是损失函数，`θ` 是参数向量。梯度中的每一项都在回答同一个问题：如果这个参数稍微改变，损失会怎样变化。

梯度的价值在于，它把高维参数空间中的局部变化压缩成一个可行动的方向信号。训练过程可以沿着梯度的反方向更新参数：

```text
θ_(t+1) = θ_t - η ∇L(θ_t)
```

其中 `η` 是学习率。这个更新式不表示系统已经知道最优解，只表示它在当前位置获得了一种可重复使用的局部修正机制。

## 4. 损失函数决定梯度在优化什么

梯度不是凭空出现的，它总是相对于某个损失函数计算出来。损失函数定义了“错”的形状，梯度则描述这个错如何随参数变化。

分类任务里，损失可能鼓励正确类别获得更高概率；回归任务里，损失可能惩罚数值偏差；语言建模里，损失会鼓励模型把概率质量分配给下一个正确 token；排序或偏好学习里，损失又会把“谁应该排在谁前面”写成训练信号。

这意味着梯度只会推动模型朝损失函数定义的方向变化。如果损失函数和真实任务目标不一致，梯度下降也可能把模型训练得更擅长优化代理目标，而不是更擅长解决真实问题。

第39章讨论过数据和评测口径如何塑造模型行为。这里可以看到另一半：数据给出样本，损失给出目标，梯度把样本上的错误转成参数更新方向。

## 5. 链式法则让深层网络可以端到端训练

深度网络由多层函数复合而成。输入经过 embedding、线性层、非线性、归一化、attention、残差连接等结构，才形成输出。损失发生在末端，但需要更新的是前面许多层的参数。

链式法则解决的正是这个传递问题。最基本形式可以写成：

```text
dz/dx = (dz/dy) · (dy/dx)
```

如果把网络看成多层复合函数，这条关系意味着：末端误差对前面某一层参数的影响，可以沿着计算路径逐层拆开、逐层相乘、逐层传回。

这就是反向传播的数学基础。反向传播不是一种神秘学习机制，而是链式法则在计算图上的系统化执行。它让深层网络可以把末端损失和前层参数连接起来，从而进行端到端训练。

## 6. 梯度只是局部方向，不是全局答案

梯度告诉系统当前位置附近的变化趋势，但它不承诺这个方向会通向全局最优。高维损失地形里有平台区、鞍点、尖锐区域、尺度差异和大量噪声，局部下降方向只是可利用的信号，不是完整地图。

这一区分很重要。训练不会因为“有了梯度”就自动完成，它还要反复使用局部方向，在许多带噪更新中逐步降低损失。每一步都只根据当前 batch、当前参数、当前数值精度和当前优化器状态做判断。

也就是说，梯度是训练语言的一部分，不是训练结果本身。它把随机搜索变成有组织的搜索，但训练能否稳定推进，还取决于学习率、优化器、batch、归一化、初始化、数值精度和数据分布。

## 7. 随机梯度让训练既可行也带噪

真实训练通常不会每一步都用完整训练集计算梯度。数据太大，代价太高，系统通常使用 mini-batch 来估计梯度。

这带来一个现实：每一步梯度都只是对整体梯度的估计。不同 batch 可能给出略有差异的方向，样本顺序、batch size、数据增强、分布式切分都会影响这个估计。

这种噪声有两面性。一方面，它会让训练曲线抖动，让更新方向不那么稳定；另一方面，它也可能帮助模型避免过早停在某些局部区域，并提供有限的探索性。

所以，工程里的梯度不是纸面上光滑、确定、一次算准的方向，而是带有采样噪声的方向估计。理解这一点，才能理解为什么 batch size、学习率调度、梯度累积和优化器状态会强烈影响训练行为。

## 8. 工程里的梯度连着显存、时间和数值稳定性

在工程实现中，梯度不仅是数学量，也是资源量。

反向传播需要保存中间激活，自动求导需要维护计算图，梯度张量需要占用显存，分布式训练需要同步梯度，混合精度训练还要处理下溢、溢出和 loss scaling。梯度能不能算出来，只是第一步；能不能稳定、经济地算出来，才是工程问题。

这也是许多训练技术出现的原因：

- activation checkpointing 用更多计算换更少显存
- gradient accumulation 用多次小 batch 累积出更大有效 batch
- gradient clipping 限制异常大梯度造成的发散
- mixed precision 提升吞吐，同时引入数值管理问题
- distributed data parallel 在多设备之间同步梯度

从这个角度看，梯度连接了一条完整链路：损失函数给出目标，自动求导给出梯度，优化器使用梯度更新参数，硬件和系统决定这一切能否在可接受成本下完成。

## 9. 这一章如何接到优化与训练

第40章解决的是“方向信号从哪里来”。下一章会进一步讨论“拿到方向之后怎样走”。

有了梯度，系统知道当前局部怎样改变参数可能降低损失；但训练还需要决定学习率多大、batch 怎么选、优化器用什么、什么时候预热、什么时候衰减、怎样判断收敛、怎样避免数值问题和资源浪费。

因此，导数和梯度是训练机制的入口，不是完整训练工程。第41章会把这种局部方向信号推进成完整的优化过程。

---

## 10. 本章收束

训练需要知道模型错了多少，也要知道错误如何随参数变化。导数刻画局部敏感度，梯度把大量偏导组织成高维参数空间中的方向信号，损失函数则决定这些方向到底在优化什么。链式法则和反向传播，把末端误差一路传回前层参数，让深层网络可以端到端训练。

但梯度只是局部方向，不是全局答案。mini-batch 梯度让训练可行，也让方向信号带有采样噪声；工程里的梯度还连着显存、时间、通信、自动求导和数值稳定性成本。深度学习能从误差中形成能力，靠的正是这种可以反复计算和利用的方向信号。
